## 二、仍存在的严重问题 🔴

### 1. **虚拟滚动未实现** 🔴 P0

**问题**:
- 虽然实现了分页查询，但**前端仍未实现虚拟滚动**
- 首页仍可能加载所有单词到 DOM
- 超过 120+ 单词时仍会卡顿

**影响**:
- 用户体验差
- 移动设备上可能崩溃

**建议**:
- 在首页组件中实现虚拟滚动（使用 `vue-virtual-scroller` 或自定义实现）
- 配合分页查询使用

---

### 2. **首日复习流程仍有问题** 🔴 P0

**问题**:
- 首日复习间隔仍然太短：
  - 第一次: 10分钟
  - 第二次: 20分钟（答错时）
  - 第三次: 6小时
  - 第四次: 24小时

**代码位置**: `learningCenter_v2.js` 第 176-213 行

**具体问题**:
```javascript
// 第一次复习只有 10 分钟，太短
first_day_due_at: new Date(now.getTime() + 10 * 60 * 1000).toISOString(),

// 答错时只有 20 分钟，不符合遗忘曲线
first_day_due_at: new Date(now.getTime() + 20 * 60 * 1000).toISOString(),
```

**影响**:
- 用户体验差：频繁的复习提醒
- 学习效率低：没有给予足够的遗忘时间

**建议方案**:
```javascript
// 改进的首日复习间隔
第一次: 15-20分钟 (给用户消化时间)
第二次: 1小时 (答错时改为30分钟)
第三次: 8小时 (答错时改为2小时)
第四次: 24小时 (答错时改为6小时)
```

---

### 3. **初始稳定性参数不合理** 🔴 P0

**问题**:
- `INITIAL_STABILITY: 0.6` (0.6天 = 14.4小时)
- 新单词首次学习后仅 14 小时就需要复习
- 不符合艾宾浩斯遗忘曲线理论

**代码位置**: `algorithmConfig.js` 第 21 行

**影响**:
- 用户需要频繁复习新单词
- 学习节奏不合理

**建议**:
```javascript
// 改为 1.5-2 天
INITIAL_STABILITY: 1.5,  // 1.5 天后首次复习
```

---

### 4. **缓存一致性问题仍未完全解决** 🟠 P1

**问题**:
- 虽然实现了缓存过期机制，但**多设备/多标签页场景下仍可能不同步**
- 缓存过期时间 (5分钟) 可能导致用户看到过时数据
- 没有实现**缓存失效通知机制**

**代码位置**: `learningCenter_v2.js` 第 24-34 行

**具体问题**:
```javascript
const CACHE_TTL_MS = 5 * 60 * 1000; // 5 分钟
// 如果用户在 A 标签页修改了单词，B 标签页的缓存在 5 分钟内不会更新
```

**影响**:
- 多设备/多标签页场景下数据不一致
- 用户修改后看不到最新数据

**建议**:
- 实现 `storage` 事件监听，当其他标签页修改数据时清除缓存
- 或使用 `BroadcastChannel` API 实现跨标签页通信
