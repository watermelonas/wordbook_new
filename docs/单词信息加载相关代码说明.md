# 单词信息加载相关代码说明

本文档说明重构后「单词列表」与「单词详情」的数据来源、加载顺序及涉及文件。路径均为**源码路径**（不含 unpackage 构建产物）。**全项目禁止在 .vue 中 import word-dict.json、pregen_vocab.json 等大 JSON**，一律按需查库或按需加载。

---

## 一、架构概览

| 数据来源 | 说明 |
|----------|------|
| **vocal_master.db** | 统一主库。表 **vocab_master**(english, chinese, data_json)，由 `trueti/build_master_db.py` 从 pregen_vocab、word-stats、word-exam-sentences、word-dict 合并生成。详情页**单词本入口**、首页释义补全、快速添加查词、复习页干扰项词表均由此库按需查询，**无大 JSON**。 |
| **pregen_data.db** | 表 **vocab**，由 `trueti/sync_pregen_to_app.py --build-db` 从 pregen_vocab.json 生成。首页非自用列表 enrich 时与 masterDb 并行使用，补全例句/同反义等；详情页单词本已优先走 masterDb。 |
| **wordbook.db** | 自用单词本。首页分页、详情页自用入口轻量/重型两阶段查询。 |
| **word-stats.json / word-exam-sentences.json** | 仅**自用详情入口**在约 1 秒后懒加载真题时使用；首页非自用列表的真题标签等由 wordStats 缓存提供（需先 loadWordStats）。**App 启动不再预热**这两个 JSON。 |

## 二、核心文件与职责

### 2.1 应用启动

| 文件 | 职责 |
|------|------|
| **App.vue** | `onLaunch` 仅执行 `db.init()`（仅 App 端），**不加载任何大 JSON、不预热 pregen/真题**。预生成与真题均在进入首页/详情时按需查库或按需加载。 |

### 2.2 首页列表

| 文件 | 职责 |
|------|------|
| **pages/index/index.vue** | **禁止 import 大 JSON**。`onLoad` 可预热 `wordStats.loadWordStats()`（可选）。自用：`db.getWordsForList(limit, offset, ...)` 分页。非自用：raw 列表 → `words = slice(0, PAGE_SIZE)` 首屏 → `enrichWordbookListInBackground(list)` 内并行 `pregenVocab.getPregenVocabCache(englishList)` 与 `masterDb.getWordBriefBatch(englishList)`，得到 cache + dictLookup，再 `enrichOneWord(list[i], cache, dictLookup)` 按批刷新。释义不足时用 dictLookup（来自 vocal_master）补全。 |
| **src/utils/db.js** | `getWordsForList(limit, offset, orderBy, orderDir, filters)` 分页；`getWordByIdLight(id)` / `getWordByIdHeavy(id)` 供详情自用两阶段。 |
| **src/utils/wordbookSource.js** | 本地单词本读 CSV、云端/自建读 storage，提供 raw 列表。 |

### 2.3 统一主库（vocal_master.db）

| 文件 | 职责 |
|------|------|
| **src/utils/masterDb.js** | 打开 `_doc/vocal_master.db`（首次从 `_www/static/vocal_master.db` 复制）。`getWordFullDetail(word)`：按 english 查一条，解析 data_json，返回 `{ chinese, examples, synonyms, antonyms, examStats, examSentences }`，容错默认值。`getWordBriefBatch(englishList)`：批量查释义，返回 `{ "english_lower": "chinese", ... }`，供首页 enrich。`getWordListForReview()`：返回词表 `[{ word, chinese }, ...]`（最多 5000 条，模块内缓存），供复习页形近/易混干扰项。 |
| **static/vocal_master.db** | 由 `python trueti/build_master_db.py` 生成；表 vocab_master，索引 idx_word。 |

### 2.4 预生成词库（pregen_data.db）

| 文件 | 职责 |
|------|------|
| **src/utils/pregenVocab.js** | `loadPregenVocab()` 已废弃。`getPregenWord(english)`、`getPregenVocabCache(englishList)` 按需打开 pregen_data.db 的 **vocab** 表查询，供首页 enrich 与原有逻辑兼容。仅 App 端有效。 |
| **static/pregen_data.db** | 由 `python trueti/sync_pregen_to_app.py --build-db` 从 pregen_vocab.json 生成。 |

### 2.5 真题数据（wordStats）

| 文件 | 职责 |
|------|------|
| **src/utils/wordStats.js** | `loadWordStats()` / `loadWordExamSentences()` 加载 word-stats.json、word-exam-sentences.json 并缓存。`getWordStatsFromCache`、`getWordExamSentencesFromCache` 同步读缓存。仅在**自用详情入口**的真题懒加载（约 1 秒后）及首页需要真题标签时使用。 |

### 2.6 单词详情页

| 文件 | 职责 |
|------|------|
| **pages/word-detail/word-detail.vue** | **禁止 import 大 JSON**。**单词本入口**：两阶段——(1) 同步展示 english、chinese: '加载中…' 及大色块占位；(2) `masterDb.getWordFullDetail(english)` 一次拉齐释义/例句/同反义/真题，赋值后 Vue 响应式更新，**不再**单独调用 loadExamDataLazy。**自用入口**：`loadWord()` 轻量查 → 重型查 examples/synonyms/antonyms → 约 1 秒后 `loadExamDataLazy(english, id)` 拉真题两 JSON。 |

### 2.7 快速添加与复习页

| 文件 | 职责 |
|------|------|
| **pages/quick-add/quick-add.vue** | **禁止 import word-dict**。查词用 `masterDb.getWordFullDetail(word.value.english)` 取释义并展示。 |
| **pages/review/review.vue** | **禁止 import word-dict**。本地单词本列表释义用 `masterDb.getWordBriefBatch(list.map(w => w.english))` 补全。形近/易混干扰项用 `masterDb.getWordListForReview()`，在 loadChoiceQuestion / loadChoiceEnQuestion / loadFillQuestion 中先 `await ensureDictWords()` 再使用。 |

## 三、数据流简图

```
【App 启动 onLaunch】
  └─ db.init()  （仅 App 端，不加载任何 JSON）

【首页 onLoad】
  ├─ 可选：wordStats.loadWordStats().then(… examStatsReady)
  └─ loadWords()
       ├─ 自用：db.getWordsForList(50, 0, …) → words 赋值
       └─ 非自用：raw → allExternalWords = raw.map(…), words = slice(0, 50)
                  → enrichWordbookListInBackground(words.value, book, words)
                       ├─ await wordStats.loadWordStats()
                       ├─ [cache, dictLookup] = await Promise.all([
                       │     pregenVocab.getPregenVocabCache(englishList),
                       │     masterDb.getWordBriefBatch(englishList)
                       │   ])
                       └─ 按批 enrichOneWord(list[i], cache, dictLookup)，刷新 words

【详情页 - 自用入口 loadWord(id)】
  ├─ await db.getWordByIdLight(id) → word 赋值，首帧展示
  ├─ db.getWordByIdHeavy(id).then(…) → 补全 examples/synonyms/antonyms
  └─ setTimeout(() => loadExamDataLazy(english, id), 1000)   // 真题 1s 后懒加载

【详情页 - 单词本入口 loadWordFromWordbook(english)】
  ├─ 首帧：word = { english, chinese: '加载中…', … }，detailHeavyLoading = true，大色块占位
  └─ masterDb.getWordFullDetail(english).then(detail) → 一次写回 chinese/examples/synonyms/antonyms/examStats/examSentences，detailHeavyLoading = false
     （不调用 loadExamDataLazy，数据已含在 detail 中）

【快速添加】searchWord → masterDb.getWordFullDetail(english) → 展示释义
【复习页】本地单词本列表 → masterDb.getWordBriefBatch(…) 补全 chinese；选项/填空干扰项 → ensureDictWords() → getWordListForReview()
```

## 四、详情页加载逻辑详解

### 4.1 自用单词本（options.id）

1. **首帧**：`await db.getWordByIdLight(id)`，只查基础字段，赋值 `word`，`incrementViewCount`、`loadSameTagWords()`。
2. **异步补全**：`db.getWordByIdHeavy(id).then(...)` 将 examples/synonyms/antonyms 合并进 `word`。
3. **真题懒加载**：约 **1 秒**后 `loadExamDataLazy(english, id)`，内部 `Promise.all([loadWordStats(), loadWordExamSentences()])`，再用 `getWordStatsFromCache`、`getWordExamSentencesFromCache` 写回 `examStats`、`examSentences`、importance、tags。

### 4.2 单词本点进（options.english + fromWordbook）

1. **首帧**：`word = { english, chinese: '加载中…', importance: 0, examples: [], synonyms: [], antonyms: [] }`，`detailHeavyLoading = true`，例句/真题/同反义区域显示**大色块占位**（`.detail-placeholder-block`）。
2. **一次拉齐**：`masterDb.getWordFullDetail(english)`，返回后写回 `word`（chinese、examples、synonyms、antonyms、importance）、`examStats`、`examSentences`、`examStatsTags`，并设 `detailHeavyLoading = false`。**不再**使用 pregenVocab 或 loadExamDataLazy，无 word-dict import。

### 4.3 详情页用到的 db 方法

| 方法 | 查询内容 | 用途 |
|------|----------|------|
| getWordByIdLight(id) | id, english, chinese, tags, source_page, year, importance, repeat_count, view_count, create_time | 自用详情首帧 |
| getWordByIdHeavy(id) | examples, synonyms, antonyms（JSON 解析后返回） | 自用详情异步补全 |
| getWordById(id) | 全字段 | 兼容旧逻辑，详情未用 |

## 五、数据库生成方式

| 库文件 | 脚本 | 说明 |
|--------|------|------|
| **static/vocal_master.db** | `python trueti/build_master_db.py` | **一劳永逸**：优先从 `trueti/kaoyan_vocab.db`（batch_ai_vocab 产出）读取例句/近义/反义，可选合并 pregen_vocab.json；再与 word-stats、word-exam-sentences、word-dict 合并。表 vocab_master(english, chinese, data_json)，含 examples/synonyms/antonyms/examStats/examSentences，索引 idx_word。无需先导出 JSON，直接运行脚本即可。 |
| **static/pregen_data.db** | `python trueti/sync_pregen_to_app.py --build-db` | 从 pregen_vocab.json 生成，表 vocab(english, chinese, examples, synonyms, antonyms)，供首页 enrich 与兼容逻辑。 |

## 六、主库更新后仍无释义/例句时

若已重新执行 `python trueti/build_master_db.py` 并生成了新的 `static/vocal_master.db`，但 App 内仍看不到释义或例句/近义词/反义词，多半是应用在使用**旧的主库副本**（首次启动时复制到 `_doc/vocal_master.db`）。处理方式任选其一：

- **清除应用数据** 或 **卸载后重装**，再打开 App，会重新从安装包复制最新的 `static/vocal_master.db`；
- 或通过文件管理器/调试删除应用私有目录下的 `vocal_master.db` 后**完全退出并重启 App**，下次启动会重新复制并打开新库。

首页释义会从 `masterDb.getWordBriefBatch` 与 `pregenVocab.getPregenVocabCache` 两路补全；详情页例句/近义/反义在主库无数据时会自动用 `pregenVocab.getPregenWord` 补全。

## 七、可优化点与代码位置速查

| 关注点 | 说明 |
|--------|------|
| 真题懒加载时机 | 自用详情为进入后约 1 秒；若需「滚动到真题区域再加载」，可改为基于滚动或 IntersectionObserver。 |
| 同标签词 | loadSameTagWords 在首帧即调用，若同标签词很多可改为进入该区块再加载或分页。 |
