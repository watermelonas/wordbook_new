# 项目深入分析：业务逻辑、功能、用户体验问题

**分析时间**: 2026-03-20
**项目**: 单词本学习应用 (wordbook_new)
**分析范围**: 业务逻辑、功能完整性、用户体验

---

## 一、业务逻辑层面的问题

### 1. **复习算法参数配置不合理** ⚠️ 高优先级

**问题位置**: `src/utils/algorithmConfig.js`

**具体问题**:
- **初始稳定性过低** (0.6天): 新单词首次学习后仅0.6天就需要复习，对用户学习节奏不友好
- **首日复习间隔设置不科学**:
  - 第一次复习: 10分钟 (太短，用户可能还在消化)
  - 第二次复习: 20分钟 (答错时，间隔太短)
  - 第三次复习: 6小时 (跨度太大，容易遗忘)
  - 第四次复习: 24小时 (可接受)

**影响**:
- 用户体验差：频繁的复习提醒
- 学习效率低：没有给予足够的遗忘时间来验证记忆
- 不符合艾宾浩斯遗忘曲线理论

**建议方案**:
```javascript
// 改进的首日复习间隔
第一次: 15分钟 (给用户消化时间)
第二次: 1小时 (答错时改为30分钟)
第三次: 8小时 (答错时改为2小时)
第四次: 24小时 (答错时改为6小时)

// 初始稳定性改为 1.5-2 天
INITIAL_STABILITY: 1.5
```

### 2. **掌握度计算逻辑缺陷** ⚠️ 中优先级

**问题位置**: `src/utils/reviewAlgo.js` 中的 `calculateMastery()` 函数

**具体问题**:
- 掌握度计算依赖 `correct_count` 和 `wrong_count`，但这两个字段在复习流程中**没有被正确更新**
- `learningCenter_v2.js` 中的复习记录只更新了 `review_count` 和 `lapse_count`，没有更新 `correct_count`
- 导致掌握度始终基于过时数据计算

**代码证据**:
```javascript
// learningCenter_v2.js 中的复习更新
const newState = scheduleReviewState(word, isCorrect);
// 这里只更新了 review_count 和 lapse_count
// 但没有更新 correct_count 和 wrong_count

// 而 calculateMastery 依赖这些字段
export const calculateMastery = (word) => {
  const correct = Number(word.correct_count || 0);
  const wrong = Number(word.wrong_count || 0);
  // ... 计算逻辑
};
```

**影响**:
- 掌握度显示不准确
- 用户无法真实了解学习进度
- 复习优先级排序错误

**修复方案**:
```javascript
// 在 scheduleReviewState 返回的状态中添加
if (isCorrect) {
  newState.correct_count = (prev.correct_count || 0) + 1;
} else {
  newState.wrong_count = (prev.wrong_count || 0) + 1;
}
```

### 3. **复习间隔计算存在边界问题** ⚠️ 中优先级

**问题位置**: `src/utils/reviewAlgo.js` 第 228-233 行

**具体问题**:
```javascript
intervalDays = clamp(
  stability * (FSRS_CONFIG.INTERVAL_BASE_FACTOR +
    (1 - difficulty) * FSRS_CONFIG.INTERVAL_DIFFICULTY_FACTOR),
  0.5,  // 最小值
  FSRS_CONFIG.INTERVAL_MAX_DAYS  // 最大值 90 天
);
```

- 最小间隔 0.5 天 = 12 小时，对于答错的单词来说太长
- 答错时应该立即复习或在几小时内复习，但当前代码设置为 `INTERVAL_ON_WRONG: 0.125` (3小时)，这个值**没有被正确应用**

**代码证据**:
```javascript
// 答错时的处理
} else {
  // ... 难度和稳定性调整
  intervalDays = FSRS_CONFIG.INTERVAL_ON_WRONG;  // 0.125 天
}
```

但这个逻辑在某些情况下会被覆盖。

**影响**:
- 答错的单词复习间隔不合理
- 用户容易再次遗忘

### 4. **首日复习流程与常规复习流程混乱** ⚠️ 高优先级

**问题位置**: `src/utils/learningCenter_v2.js` 中的 `recordReview()` 函数

**具体问题**:
- 首日复习 (`first_day_stage`) 和常规复习 (`scheduleReviewState`) 是两套独立的系统
- 当首日复习完成后，转换到常规复习时，**初始参数设置不当**
- 首日复习完成后的单词，其 `stability` 和 `difficulty_score` 可能不合理

**代码证据**:
```javascript
// 首日复习完成后
if (stage === 4) {
  return {
    first_day_stage: 4,
    first_day_due_at: '',
    first_learned_at: profile.first_learned_at,
  };
  // 这里没有初始化 stability 和 difficulty_score
}
```

**影响**:
- 首日复习完成后的单词进入常规复习时，参数不一致
- 可能导致复习间隔计算错误

### 5. **缓存一致性问题** ⚠️ 中优先级

**问题位置**: `src/utils/learningCenter_v2.js` 中的缓存管理

**具体问题**:
- 使用了两层缓存（内存 + 本地存储），但**缓存失效机制不完善**
- 当用户在多个设备或标签页操作时，缓存可能不同步
- 缓存过期时间 (5分钟) 可能导致用户看到过时数据

**代码证据**:
```javascript
const CACHE_TTL_MS = 5 * 60 * 1000; // 5 分钟
const STORAGE_TTL_MS = 24 * 60 * 60 * 1000; // 24 小时

// 问题：如果用户在 A 标签页修改了单词，B 标签页的缓存在 5 分钟内不会更新
```

**影响**:
- 多设备/多标签页场景下数据不一致
- 用户修改后看不到最新数据

### 6. **单词去重逻辑不完善** ⚠️ 中优先级

**问题位置**: `src/utils/learningCenter_v2.js` 中的 `normalizeWordKey()`

**具体问题**:
```javascript
export const normalizeWordKey = (word) => {
  if (!word) return '';
  if (typeof word === 'string') return word.trim().toLowerCase();
  return String(word.english || '').trim().toLowerCase();
};
```

- 只考虑了英文单词，没有考虑**同义词、短语、变体形式**
- 例如: "run", "runs", "running", "ran" 会被视为不同的单词
- 没有处理**特殊字符和空格**

**影响**:
- 同一个单词的不同形式被视为不同单词
- 学习数据分散，掌握度计算不准确

## 四、数据一致性问题

### 1. **多表数据同步问题** ⚠️ 高优先级

**问题位置**: `src/utils/db_v2.js`

**具体问题**:
- 有 `words` 表和 `mastered_words` 表
- 当单词从 `words` 移到 `mastered_words` 时，**没有事务保护**
- 可能导致数据重复或丢失

**代码证据**:
```javascript
// 没有事务保护
async addWord(word) {
  // 添加到 words 表
}

async markAsMastered(id) {
  // 从 words 表删除，添加到 mastered_words 表
  // 但这两个操作不是原子的
}
```

**影响**:
- 数据可能重复或丢失
- 用户数据不可靠

---

### 2. **本地存储与数据库不同步** ⚠️ 中优先级

**问题**:
- 学习档案存储在本地存储中 (`learningCenter_v2.js`)

## 九、总结

该项目在以下方面存在显著问题：

1. **业务逻辑**: 复习算法参数不合理，掌握度计算错误，缓存一致性差
2. **功能完整性**: 虚拟滚动未实现，云端同步不清晰，导入导出不完整
3. **用户体验**: 掌握度显示不准确，复习提醒不及时，列表性能差
4. **数据一致性**: 多表同步问题，缓存与数据库不同步
5. **性能**: 大数据集加载慢，数据库查询未优化，内存泄漏风险

**建议**: 按照优先级排序，先修复 P0 问题，再逐步改进 P1 和 P2 问题。
